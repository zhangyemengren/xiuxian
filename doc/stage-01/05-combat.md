# 步骤 05：战斗结算

## 目标

让战斗事件产生可解释的胜负、生命值变化以及奖励或惩罚。

## 依赖

- 步骤 04：随机事件。

## 基础规则

```text
玩家攻击 = 灵力属性 + 随机波动
怪物攻击 = 等级基础值 + 随机波动
```

- 玩家攻击不低于怪物攻击时，战斗胜利；
- 玩家攻击低于怪物攻击时，战斗失败；
- 战斗会造成生命值损失；
- 胜利获得经验或资源；
- 失败扣除一定经验或资源；
- 生命值最低为 `0`。

具体波动范围、伤害和奖惩数值由配置管理。

## 范围

- 根据角色等级生成基础怪物；
- 计算双方攻击值和战斗结果；
- 应用生命值变化；
- 应用胜利奖励或失败惩罚；
- 输出完整战斗日志。

## 非目标

- 多回合战斗指令；
- 技能、装备和状态效果；
- 怪物图鉴；
- 角色死亡后的完整处理流程。

## 验收标准

- 每个战斗事件恰好产生一次结算；
- 结算日志包含双方攻击、胜负、生命变化和奖惩；
- 生命值、经验和资源不会出现负数；
- 奖励和惩罚与配置一致；
- 固定输入和随机源时结算结果可重复。

## 实现方向

```rust
struct Monster {
    level: u32,
    attack_base: u32,
}

enum CombatOutcome {
    Victory,
    Defeat,
}

struct CombatResult {
    player_attack: u32,
    monster_attack: u32,
    hp_lost: u32,
    outcome: CombatOutcome,
    experience_delta: i32,
    spirit_stones_delta: i32,
    herbs_delta: i32,
}
```

先计算战斗结果，再把变化安全地应用到角色。所有攻击波动、伤害和奖惩集中配置；本步骤不处理完整死亡流程。
