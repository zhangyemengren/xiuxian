# 步骤 02：角色状态

## 目标

为自动模拟增加角色基础属性、状态和资源，并在每回合结束时输出当前状态。

## 依赖

- 步骤 01：基础循环。

## 初始状态

| 属性 | 初始值 |
| --- | ---: |
| 生命值（HP） | 100 |
| 灵力值（MP） | 50 |
| 体质 | 1～5 随机值 |
| 灵力属性 | 1～5 随机值 |
| 悟性 | 1～5 随机值 |
| 等级 | 1 |
| 经验 | 0 |
| 灵石 | 0 |
| 草药 | 0 |

## 范围

- 模拟开始时创建一个角色；
- 保存角色属性、等级、经验和资源；
- 每回合结束输出角色状态；
- 支持固定初始值，以便测试。

## 非目标

- 属性对修炼和战斗的影响；
- 属性变化；
- 升级。

## 验收标准

- 角色初始状态符合表格定义；
- 随机基础属性始终位于 `1～5`；
- 状态日志包含 HP、MP、等级、经验、灵石和草药；
- 本步骤中各回合状态保持不变；
- 固定初始值时输出结果可重复。

## 实现方向

```rust
struct Attributes {
    constitution: u32,
    spirit: u32,
    comprehension: u32,
}

struct Resources {
    spirit_stones: u32,
    herbs: u32,
}

struct Character {
    hp: u32,
    max_hp: u32,
    mp: u32,
    max_mp: u32,
    level: u32,
    experience: u32,
    attributes: Attributes,
    resources: Resources,
}
```

- “灵力值”和“灵力属性”使用不同字段；
- 当前值不能超过最大值；
- 资源扣减使用安全运算；
- 随机初始化使用模拟器传入的随机源；
- 主要影响 `character`、`simulation` 和 `time_line` 模块。
