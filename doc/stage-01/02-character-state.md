# 步骤 02：角色状态

## 目标

为自动模拟增加角色基础属性、状态和资源，并在每回合结束时输出当前状态。

## 依赖

- 步骤 01：基础循环。

## 初始状态

| 属性 | 初始值 |
| --- | ---: |
| 生命值（HP） | 100 |
| 灵力值（MP） | 50 |
| 体质 | 1～5 随机值 |
| 灵力属性 | 1～5 随机值 |
| 悟性 | 1～5 随机值 |
| 等级 | 1 |
| 经验 | 0 |
| 灵石 | 0 |
| 草药 | 0 |

## 范围

- 模拟开始时创建一个角色；
- 保存角色属性、等级、经验和资源；
- 每回合结束输出角色状态；
- 支持固定初始值，便于人工检查运行结果。

## 非目标

- 属性对修炼和战斗的影响；
- 属性变化；
- 升级。

## 业务规则

- “灵力值”和“灵力属性”是两个不同概念；
- 当前生命值不能超过生命上限；
- 当前灵力值不能超过灵力上限；
- 生命值、灵力值、经验和资源不能出现负数；
- 同一次模拟中，角色初始状态创建后保持一致；
- 固定初始值时，每次运行应得到相同结果。
