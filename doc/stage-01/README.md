# 第一阶段：自动修炼

## 要做成什么

程序在没有玩家输入的情况下，自动完成修炼、随机事件、战斗和角色成长，最后输出时间线和统计结果。

第一阶段暂时不做玩家输入、图形界面、存档、副本、宗门和复杂战斗。

## 程序结构方向

```text
src/
├── main.rs          # 程序入口
├── app.rs           # 组装并启动模拟
├── simulation/      # 回合循环和配置
├── character/       # 角色状态
├── cultivation/     # 修炼规则
├── event/           # 随机事件
├── combat/          # 战斗结算
├── growth/          # 等级成长
├── statistics/      # 汇总统计
├── time_line/       # 时间线记录和显示
└── world/           # 世界时间和后续世界规则
```

不需要提前创建所有模块，做到哪个步骤再创建哪个。

## 一回合的最终流程

```text
开始回合
  -> 修炼
  -> 随机事件
  -> 必要时战斗
  -> 检查升级
  -> 记录角色状态和统计
结束回合
```

## 共同约定

- 规则计算返回结构化结果，日志只负责显示结果。
- 概率、经验、资源和成长数值集中配置。
- `Simulation` 统一持有随机源，各模块不私自创建随机源。
- 同一个配置和随机种子必须产生相同结果。
- 每个步骤完成后，程序仍然可以独立运行和测试。

## 阶段完成标准

- 七个步骤全部完成；
- 可以连续模拟至少 `100` 回合；
- 相同随机种子可以复现相同结果；
- 日志中的行为和状态变化前后一致；
- 汇总数据可以用于检查事件、战斗、成长和资源趋势。
