# 第一阶段：自动修炼

## 要做成什么

程序在没有玩家输入的情况下，自动完成修炼、随机事件、战斗和角色成长，最后输出时间线和统计结果。

第一阶段暂时不做玩家输入、图形界面、存档、副本、宗门和复杂战斗。

## 单回合业务流程

1. 检查是否达到停止条件；
2. 开始回合；
3. 推进并输出世界时间；
4. 角色修炼；
5. 判断是否触发随机事件；
6. 必要时进行战斗；
7. 检查角色是否升级或死亡；
8. 记录角色状态和统计；
9. 结束回合并继续下一次循环。

## 共同规则

- 概率、经验、资源和成长数值应当可以统一调整；
- 相同配置和随机种子必须产生相同结果；
- 每个步骤完成后，程序仍然可以独立运行；
- 日志中的结果必须与角色实际状态一致。
- 各功能负责输出自己的日志，应用入口只负责启动流程。

## 阶段完成标准

- 七个步骤全部完成；
- 可以连续模拟至少 `100` 回合；
- 相同随机种子可以复现相同结果；
- 日志中的行为和状态变化前后一致；
- 汇总数据可以用于检查事件、战斗、成长和资源趋势。
