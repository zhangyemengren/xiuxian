# 步骤 07：统计验证

## 目标

汇总完整模拟过程中的关键数据，为第一阶段的数值验证提供依据。

## 依赖

- 步骤 06：等级成长。

## 汇总指标

- 总回合数；
- 各修炼类型执行次数；
- 各事件类型触发次数和占比；
- 总战斗次数、胜利次数和胜率；
- 灵石与草药的获得量和最终持有量；
- 升级次数；
- 初始与最终属性；
- HP、MP、经验和资源的变化趋势。

## 范围

- 每回合结束后采集结构化指标；
- 模拟结束后输出汇总报告；
- 对没有发生过的行为安全输出 `0`；
- 报告包含本次模拟配置和随机种子。

## 非目标

- 图形化报表；
- 数据库存储；
- 多次模拟之间的长期分析；
- 自动调整平衡参数。

## 验收标准

- 汇总值与时间线逐条计算的结果一致；
- 战斗次数为 `0` 时胜率不会出现除零错误；
- 事件次数总和与事件判定次数一致；
- 初始值、最终值和累计变化可以相互校验；
- `100` 回合模拟能够完整输出日志和汇总报告。

## 实现方向

```rust
struct SimulationStatistics {
    rounds: u32,
    cultivation_counts: CultivationCounts,
    event_counts: EventCounts,
    battles: u32,
    victories: u32,
    level_ups: u32,
    resources_gained: Resources,
    snapshots: Vec<StateSnapshot>,
}
```

统计模块读取前面步骤返回的结构化结果，不解析日志文本。模拟结束后先生成纯数据报告，再格式化为文本；所有比例都必须处理分母为 `0` 的情况。
