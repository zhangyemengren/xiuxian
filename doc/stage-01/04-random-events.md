# 步骤 04：随机事件

## 目标

在修炼后按配置概率触发事件，并确保事件序列可以复现和统计。

## 依赖

- 步骤 03：修炼系统。

## 事件类型

| 事件 | 结果 |
| --- | --- |
| 无事件 | 不产生额外变化 |
| 宝物 | 获得灵石或草药 |
| 修炼失败 | 撤销本回合修炼收益 |
| 战斗 | 产生战斗事件，具体结算由后续步骤实现 |

## 范围

- 为每种事件配置触发权重或概率；
- 每次修炼后最多触发一个事件；
- 宝物事件增加资源；
- 修炼失败事件撤销本回合经验收益；
- 战斗事件先记录为“待结算”；
- 支持固定随机种子。

## 非目标

- 战斗胜负结算；
- 高级事件和事件链；
- 玩家事件选择。

## 验收标准

- 所有事件概率配置合法且总和可校验；
- 单回合不会同时触发多个互斥事件；
- 宝物和修炼失败正确改变角色状态；
- 相同配置和随机种子产生相同事件序列；
- 大样本运行时可统计每类事件次数。

## 实现方向

```rust
enum EventKind {
    None,
    Treasure,
    CultivationFailure,
    Combat,
}

enum EventResult {
    None,
    Treasure { spirit_stones: u32, herbs: u32 },
    CultivationFailure { experience_reverted: u32 },
    CombatPending,
}
```

随机种子属于模拟配置，随机数生成器由 `Simulation` 统一持有。事件模块只使用传入的随机源；权重选择逻辑需要单独测试。
