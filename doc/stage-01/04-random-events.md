# 步骤 04：随机事件

## 目标

在修炼后按配置概率触发事件，并确保事件序列可以复现和统计。

## 依赖

- 步骤 03：修炼系统。

## 事件类型

| 事件 | 结果 |
| --- | --- |
| 无事件 | 不产生额外变化 |
| 宝物 | 获得灵石或草药 |
| 修炼失败 | 撤销本回合修炼收益 |
| 战斗 | 产生战斗事件，具体结算由后续步骤实现 |

## 范围

- 为每种事件配置触发权重或概率；
- 每次修炼后最多触发一个事件；
- 宝物事件增加资源；
- 修炼失败事件撤销本回合经验收益；
- 战斗事件先记录为“待结算”；
- 支持固定随机种子。

## 非目标

- 战斗胜负结算；
- 高级事件和事件链；
- 玩家事件选择。

## 业务规则

1. 每次修炼完成后进行一次事件判断。
2. 每次判断最多产生一个事件。
3. 宝物事件增加灵石或草药。
4. 修炼失败事件撤销本回合的修炼收益。
5. 战斗事件在本步骤只记录为待结算。
6. 相同配置和随机种子必须产生相同事件序列。
